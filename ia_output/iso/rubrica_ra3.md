# Rúbrica — RA3: Asegura y recupera el sistema, aplicando procedimientos de administración de la información.

**Módulo:** Implantación de Sistemas Operativos  
**Ponderación:** 9%

| Criterio | No Alcanzado (0-4) | En Proceso (5-6) | Alcanzado (7-8) | Sobresaliente (9-10) |
|----------|---------------------|-------------------|------------------|-----------------------|
| CR1 | No se identifican los componentes del sistema operativo necesarios para el proceso de recuperación. | Se identifican algunos componentes, pero no se comprenden sus funciones en la recuperación del sistema. | Se identifican correctamente todos los componentes necesarios y se explican sus funciones. | Se identifican y analizan en profundidad todos los componentes, incluyendo sus interacciones en la recuperación del sistema. |
| CR2 | No se aplican los procedimientos de copia de seguridad, o se realizan de manera inadecuada. | Se aplican algunos procedimientos de copia de seguridad, pero con errores o falta de consistencia. | Se aplican correctamente los procedimientos de copia de seguridad, garantizando la integridad de la información. | Se aplican de manera óptima y se documentan los procedimientos de copia de seguridad, demostrando un enfoque proactivo. |
| CR3 | No se realizan pruebas de restauración o se realizan de manera incorrecta. | Se realizan pruebas de restauración, pero no se documentan adecuadamente los resultados. | Se realizan pruebas de restauración de forma efectiva y se documentan los resultados de manera clara. | Se realizan pruebas exhaustivas de restauración y se generan informes detallados con análisis de riesgos y mejoras sugeridas. |
| CR4 | No se configura el sistema para la recuperación ante desastres. | Se configura el sistema de manera básica, pero con fallos en la implementación. | Se configura correctamente el sistema para recuperación ante desastres, cumpliendo con las mejores prácticas. | Se configura el sistema con soluciones avanzadas para recuperación ante desastres, optimizando tiempo y recursos. |
| CR5 | No se aplican políticas de seguridad en la gestión de la información. | Se aplican algunas políticas de seguridad, pero no son coherentes o completas. | Se aplican políticas de seguridad de manera efectiva en la gestión de la información. | Se diseñan e implementan políticas de seguridad innovadoras y personalizadas basadas en análisis de riesgos. |
| CR6 | No se documentan los procedimientos de administración de la información. | Se documentan parcialmente los procedimientos, pero carecen de claridad o detalle. | Se documentan de forma precisa todos los procedimientos de administración de la información. | Se documentan exhaustivamente todos los procedimientos, incluyendo análisis de impacto y recomendaciones. |
| CR7 | No se realizan copias de seguridad de forma programada o no se cumple con el calendario establecido. | Se realizan copias de seguridad programadas, pero con fallos en la frecuencia o en la integridad. | Se realizan copias de seguridad programadas de manera efectiva y se cumplen los plazos establecidos. | Se implementan estrategias avanzadas de copias de seguridad, optimizando el uso del tiempo y los recursos disponibles. |
| CR8 | No se evalúa la eficacia de los procedimientos de recuperación aplicados. | Se evalúan algunos procedimientos de recuperación, pero de forma superficial. | Se evalúan de manera crítica todos los procedimientos de recuperación aplicados, generando resultados útiles. | Se lleva a cabo una evaluación exhaustiva y se proponen mejoras basadas en análisis de datos y resultados. |
| **NOTA GLOBAL** | No se ha demostrado un conocimiento básico sobre la administración de sistemas y recuperación de información. | Se ha demostrado un entendimiento parcial, aunque con varias áreas que requieren mejora y desarrollo. | Se ha alcanzado un nivel adecuado de competencia en la administración y recuperación de sistemas. | Se ha demostrado un nivel de excelencia en la administración y recuperación de sistemas, con propuestas innovadoras y un enfoque profesional. |
