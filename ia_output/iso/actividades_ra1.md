# Propuestas de Actividades — RA1: Instala sistemas operativos, relacionando sus características con el hardware del equipo y el software de aplicación.

**Módulo:** Implantación de Sistemas Operativos

### Actividad 1: "Instalación de un Sistema Operativo en un Servidor"

1. **Nombre:** Instalación de un SO en Servidor
2. **Objetivo:** Aprender a instalar un sistema operativo en un servidor, relacionando características del hardware con el software.
3. **Descripción:** Los alumnos se dividirán en grupos y recibirán un servidor físico o virtual. Cada grupo deberá investigar y seleccionar un sistema operativo adecuado para el hardware disponible. Luego, realizarán la instalación del sistema operativo, asegurándose de que los controladores y configuraciones del hardware estén correctamente instalados. Al final, cada grupo presentará un informe que detalle el proceso de instalación y justifique las decisiones tomadas. 
4. **Criterios cubiertos:** CR1, CR2, CR3, CR4, CR5, CR6.
5. **Instrumento de evaluación:** Proyecto.
6. **Duración estimada:** 6 horas.
7. **Recursos necesarios:** Servidores físicos o virtuales, medios de instalación del SO (USB, imagen ISO), red de conexión, documentación de instalación.

### Actividad 2: "Configuración de un Entorno de Trabajo Virtual"

1. **Nombre:** Entorno Virtual de Trabajo
2. **Objetivo:** Comprender la instalación y configuración de un sistema operativo en entornos virtuales, y su relación con el hardware virtualizado.
3. **Descripción:** Los estudiantes utilizarán un software de virtualización (como VirtualBox o VMware) para crear máquinas virtuales. Deberán instalar un sistema operativo en cada máquina virtual y configurar las especificaciones de hardware (RAM, disco duro, CPU) según los requisitos del sistema operativo elegido. Al finalizar, presentarán un documento que describa el proceso de instalación, la configuración realizada y los problemas encontrados durante la práctica, junto con sus soluciones.
4. **Criterios cubiertos:** CR1, CR2, CR3, CR4, CR5, CR7.
5. **Instrumento de evaluación:** Práctica.
6. **Duración estimada:** 4 horas.
7. **Recursos necesarios:** Computadoras con capacidad para virtualización, software de virtualización, imágenes ISO de sistemas operativos.

### Actividad 3: "Análisis Comparativo de Sistemas Operativos"

1. **Nombre:** Comparativa de Sistemas Operativos
2. **Objetivo:** Evaluar diferentes sistemas operativos en función de sus características y compatibilidad con hardware y software de aplicación.
3. **Descripción:** En esta actividad, los alumnos investigarán y compararán al menos tres sistemas operativos diferentes (por ejemplo, Windows, Linux y MacOS) en términos de requisitos de hardware, funcionalidades, y compatibilidad con software de aplicación. Deberán elaborar una presentación que incluya gráficos y tablas comparativas, y justificar cuál sería el sistema operativo más adecuado para diferentes entornos laborales (oficina, servidor, desarrollo, etc.). Al final, se realizará una exposición en clase.
4. **Criterios cubiertos:** CR1, CR2, CR3, CR5, CR6, CR8.
5. **Instrumento de evaluación:** Proyecto.
6. **Duración estimada:** 5 horas.
7. **Recursos necesarios:** Acceso a Internet, software de presentación, materiales para gráficos (papel, marcadores, etc.).
