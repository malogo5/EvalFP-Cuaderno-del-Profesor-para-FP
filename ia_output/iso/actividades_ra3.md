# Propuestas de Actividades — RA3: Asegura y recupera el sistema, aplicando procedimientos de administración de la información.

**Módulo:** Implantación de Sistemas Operativos

### Actividad 1: **Simulación de Recuperación de Sistema**

1. **Objetivo:** Aprender a aplicar procedimientos de recuperación de un sistema operativo en caso de fallos.
2. **Descripción:** En esta actividad, los estudiantes trabajarán en grupos para simular un escenario laboral en el que un servidor ha sufrido un fallo crítico. Usarán un entorno virtualizado para restaurar el sistema a partir de una copia de seguridad. Cada grupo recibirá un caso específico que incluye datos del sistema y las configuraciones previas. Los estudiantes deberán documentar cada paso del proceso de recuperación, desde el diagnóstico del problema hasta la restauración completa del servicio. El entregable será un informe detallado que incluya las decisiones tomadas y los comandos utilizados.
3. **Criterios cubiertos:** CR1, CR2, CR3, CR4, CR5.
4. **Instrumento de evaluación:** Proyecto.
5. **Duración estimada:** 4 horas.
6. **Recursos necesarios:** Software de virtualización (como VirtualBox), imágenes de sistemas operativos, herramientas de copia de seguridad y restauración, documentación técnica.

### Actividad 2: **Creación de un Plan de Copias de Seguridad**

1. **Objetivo:** Diseñar un plan de copias de seguridad efectivo para un entorno organizacional.
2. **Descripción:** Los estudiantes trabajarán en parejas para crear un plan de copias de seguridad para una empresa ficticia con diferentes tipos de datos y sistemas críticos. Deberán analizar las necesidades de respaldo según la información proporcionada, elegir la estrategia adecuada (completa, incremental, diferencial) y seleccionar las herramientas más adecuadas. La actividad culminará con una presentación del plan, justificando las elecciones hechas y los procedimientos de recuperación asociados. El entregable será un documento que incluya el plan completo y la presentación.
3. **Criterios cubiertos:** CR2, CR6, CR7, CR8.
4. **Instrumento de evaluación:** Proyecto.
5. **Duración estimada:** 3 horas.
6. **Recursos necesarios:** Plantillas de planes de copias de seguridad, software para gestión de copias de seguridad (por ejemplo, Acronis, Veeam), acceso a Internet para investigación.

### Actividad 3: **Auditoría de Seguridad de Datos**

1. **Objetivo:** Evaluar la seguridad de los datos y la efectividad de los procedimientos de recuperación en un sistema.
2. **Descripción:** Los estudiantes realizarán una auditoría de seguridad en un entorno de sistema operativo simulado, donde se les proporcionará un sistema con diversas configuraciones y potenciales vulnerabilidades. Deberán identificar las debilidades en los procedimientos de administración de la información, ejecutar pruebas de recuperación y sugerir mejoras. El entregable será un informe de auditoría que incluya hallazgos, procedimientos de recuperación evaluados y recomendaciones para mejorar la seguridad. 
3. **Criterios cubiertos:** CR1, CR2, CR5, CR6, CR8.
4. **Instrumento de evaluación:** Proyecto.
5. **Duración estimada:** 5 horas.
6. **Recursos necesarios:** Herramientas de auditoría de seguridad (como Wireshark, Nessus), acceso a un entorno de laboratorio de sistemas operativos, documentación de buenas prácticas de seguridad.
