# Rúbrica — RA4: Gestiona dominios, elaborando el modelo lógico de la red en un entorno de trabajo.

**Módulo:** Implantación de Sistemas Operativos  
**Ponderación:** 16%

| Criterio | No Alcanzado (0-4) | En Proceso (5-6) | Alcanzado (7-8) | Sobresaliente (9-10) |
|----------|---------------------|------------------|-----------------|----------------------|
| CR1      | No identifica los componentes de red necesarios para la gestión de dominios. | Identifica algunos componentes, pero falta claridad en su relación con el modelo lógico. | Identifica correctamente los componentes de red y su función en el modelo lógico. | Identifica y explica detalladamente todos los componentes de red, incluyendo su interrelación. |
| CR2      | No elabora un modelo lógico de red; falta coherencia y estructura. | Presenta un modelo lógico básico, pero con errores en la representación gráfica y falta de detalles. | Elabora un modelo lógico claro y coherente, con representación gráfica adecuada y correcta. | El modelo lógico es excepcional, incluye aspectos avanzados y se presenta de forma profesional. |
| CR3      | No establece las relaciones entre los diferentes elementos del dominio. | Establece algunas relaciones, pero son confusas o incompletas. | Establece claramente las relaciones entre los elementos del dominio, mostrando lógica y coherencia. | Las relaciones son complejas y bien fundamentadas, demostrando un alto entendimiento del dominio. |
| CR4      | No utiliza herramientas de software para la creación del modelo lógico. | Utiliza herramientas de software, pero con limitaciones en su aplicación y resultados. | Utiliza adecuadamente herramientas de software para crear un modelo lógico funcional y estético. | Utiliza herramientas de manera avanzada, incorporando funcionalidades que mejoran el modelo lógico. |
| CR5      | No documenta el proceso de creación del modelo lógico ni las decisiones tomadas. | Documenta parcialmente el proceso, pero con falta de claridad y detalles. | Documenta de manera clara y completa el proceso de creación del modelo lógico y las decisiones tomadas. | La documentación es exhaustiva y refleja una profunda reflexión sobre el proceso y las decisiones. |
| CR6      | No considera aspectos de seguridad en el diseño del dominio. | Considera algunos aspectos de seguridad, pero de forma superficial. | Considera aspectos de seguridad relevantes en el modelo lógico, demostrando comprensión de su importancia. | Integra de forma excepcional aspectos de seguridad, anticipando posibles vulnerabilidades y soluciones. |
| CR7      | No presenta un análisis de las necesidades del entorno de trabajo. | Realiza un análisis básico, pero carece de profundidad y detalle. | Realiza un análisis completo y detallado de las necesidades del entorno de trabajo. | El análisis es exhaustivo y considera aspectos innovadores y proactivos para el entorno de trabajo. |
| CR8      | No justifica las decisiones tomadas en el modelo lógico. | Justifica algunas decisiones, pero con argumentos débiles o poco claros. | Justifica las decisiones de manera coherente y lógica, mostrando comprensión de su impacto. | Justifica de forma brillante las decisiones, con argumentos sólidos y evidencia que respalda cada elección. |
| **NOTA GLOBAL** | El alumno muestra una comprensión limitada del manejo de dominios y elaboración del modelo lógico, con múltiples áreas de mejora. | El alumno está en proceso de comprensión, presentando algunos aciertos, pero con importantes falencias. | El alumno demuestra una comprensión adecuada, elaborando un modelo lógico coherente y bien estructurado. | El alumno alcanza un nivel sobresaliente, evidenciando un dominio completo del tema con un modelo lógico excepcional y justificaciones sólidas. |
