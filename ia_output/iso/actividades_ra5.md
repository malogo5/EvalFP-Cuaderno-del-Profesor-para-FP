# Propuestas de Actividades — RA5: Administra el acceso al dominio, gestionando servidores de directorio y verificando el acceso de los usuarios.

**Módulo:** Implantación de Sistemas Operativos

### Actividad 1: Configuración de un Servidor de Directorio

1. **Nombre:** Configuración de un Servidor de Directorio
2. **Objetivo:** El alumnado aprenderá a instalar y configurar un servidor de directorio (como Active Directory) y gestionar usuarios y grupos.
3. **Descripción:** Los estudiantes se situarán en un entorno simulado de una empresa que requiere la creación de un dominio para gestionar sus recursos. Deberán instalar un servidor Windows Server, configurar Active Directory, crear OU (Unidades Organizativas) y gestionar cuentas de usuario. El entregable será un informe técnico que documente el proceso de configuración y las decisiones tomadas, así como capturas de pantalla de la configuración realizada.
4. **Criterios cubiertos:** CR1, CR2, CR5, CR6
5. **Instrumento de evaluación:** Proyecto
6. **Duración estimada:** 8 horas
7. **Recursos necesarios:** Ordenadores con Windows Server, software de virtualización (como VMware o VirtualBox), acceso a internet, documentos de referencia sobre Active Directory.

### Actividad 2: Gestión de Políticas de Grupo (GPO)

1. **Nombre:** Implementación de Políticas de Grupo
2. **Objetivo:** Los alumnos aprenderán a crear y aplicar Políticas de Grupo para gestionar el acceso y la configuración de los usuarios en el dominio.
3. **Descripción:** En un entorno de laboratorio, los alumnos deberán aplicar diferentes políticas de grupo a los usuarios y equipos de un dominio previamente configurado. Se les proporcionará un caso práctico donde deberán establecer restricciones de acceso a aplicaciones, configuraciones de escritorio y seguridad. El entregable será un documento que explique las políticas aplicadas y un análisis de su impacto en los usuarios.
4. **Criterios cubiertos:** CR3, CR4, CR7, CR8
5. **Instrumento de evaluación:** Práctica
6. **Duración estimada:** 6 horas
7. **Recursos necesarios:** Ordenadores con Windows Server y clientes Windows, software de gestión de políticas de grupo (GPO), documentación sobre políticas de grupo.

### Actividad 3: Auditoría de Accesos y Seguridad

1. **Nombre:** Auditoría de Accesos en el Dominio
2. **Objetivo:** Los alumnos aprenderán a realizar auditorías de acceso y gestionar registros de seguridad para verificar el acceso de usuarios a los recursos del dominio.
3. **Descripción:** En esta actividad, se presentará un escenario realista donde se sospecha de accesos no autorizados en el sistema. Los estudiantes deberán habilitar el registro de auditoría en Active Directory, analizar los logs de seguridad y generar un informe que resuma los accesos realizados, identificando posibles brechas de seguridad. El entregable será un informe detallado con las acciones recomendadas para mejorar la seguridad.
4. **Criterios cubiertos:** CR2, CR5, CR6, CR8
5. **Instrumento de evaluación:** Proyecto
6. **Duración estimada:** 5 horas
7. **Recursos necesarios:** Ordenadores con Windows Server, acceso a herramientas de análisis de logs (como Event Viewer), documentación sobre auditoría de seguridad en Active Directory.
