# Propuestas de Actividades — RA4: Gestiona dominios, elaborando el modelo lógico de la red en un entorno de trabajo.

**Módulo:** Implantación de Sistemas Operativos

### Actividad 1: Diseño de un Modelo Lógico de Red

1. **Nombre:** Diseño de un Modelo Lógico de Red
2. **Objetivo:** Aprender a diseñar un modelo lógico de red acorde a las necesidades de una empresa ficticia.
3. **Descripción:** Los estudiantes trabajarán en grupos para crear un modelo lógico de red para una empresa de tamaño medio que busca modernizar su infraestructura. Deberán analizar los requerimientos de la empresa, incluyendo número de usuarios, tipos de dispositivos y servicios que se desean implementar. Utilizando herramientas de diagramación, cada grupo elaborará un diagrama técnico que represente la estructura de la red, incluyendo servidores, estaciones de trabajo, y dispositivos de seguridad. El entregable será un documento que incluya tanto el diagrama como una breve justificación de las decisiones tomadas.
4. **Criterios cubiertos:** CR1, CR2, CR3, CR4, CR5
5. **Instrumento de evaluación:** Proyecto
6. **Duración estimada:** 6 horas
7. **Recursos necesarios:** Software de diagramación (por ejemplo, Microsoft Visio o Lucidchart), acceso a internet para investigación, y materiales de oficina.

### Actividad 2: Configuración de un Dominio Active Directory

1. **Nombre:** Configuración de un Dominio Active Directory
2. **Objetivo:** Adquirir habilidades en la configuración y gestión de un dominio utilizando Active Directory.
3. **Descripción:** En esta actividad, los estudiantes recibirán un caso práctico en el que se les solicita configurar un dominio de Active Directory para una empresa. Deberán instalar el servidor, crear las unidades organizativas, y añadir usuarios y grupos. Durante el proceso, se les guiará para que entiendan los roles y servicios implicados. Al finalizar, los estudiantes presentarán su configuración y explicarán las funciones de cada uno de los elementos creados. El entregable será un informe técnico y una demostración práctica.
4. **Criterios cubiertos:** CR1, CR2, CR6, CR7, CR8
5. **Instrumento de evaluación:** Práctica
6. **Duración estimada:** 8 horas
7. **Recursos necesarios:** Servidor virtual con Windows Server, acceso a software de virtualización (como VMware o VirtualBox), y manuales de configuración.

### Actividad 3: Implementación de Políticas de Seguridad en un Dominio

1. **Nombre:** Implementación de Políticas de Seguridad en un Dominio
2. **Objetivo:** Comprender y aplicar políticas de seguridad en un entorno de red basado en un dominio.
3. **Descripción:** En esta actividad, los estudiantes trabajarán en grupos para implementar políticas de seguridad en un dominio existente. Se les proporcionará un entorno simulado donde deberán aplicar políticas de contraseñas, restricciones de acceso y auditoría. Además, deberán crear un plan de contingencia en caso de violaciones de seguridad. El entregable será un reporte que detalle las políticas implementadas y un análisis de cómo estas mejoran la seguridad del dominio.
4. **Criterios cubiertos:** CR3, CR4, CR5, CR6, CR8
5. **Instrumento de evaluación:** Proyecto
6. **Duración estimada:** 6 horas
7. **Recursos necesarios:** Acceso a un entorno de Active Directory, documentación sobre políticas de seguridad, y materiales de oficina.
