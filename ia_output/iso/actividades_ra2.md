# Propuestas de Actividades — RA2: Gestiona el software de base, aplicando criterios de explotación del sistema.

**Módulo:** Implantación de Sistemas Operativos

### Actividad 1: Instalación y Configuración de un Servidor LAMP
1. **Nombre**: Instalación de Servidor LAMP
2. **Objetivo**: Aprender a instalar y configurar un servidor web utilizando el stack LAMP (Linux, Apache, MySQL, PHP).
3. **Descripción**: En un entorno simulado que representa una pequeña empresa, los alumnos recibirán la tarea de instalar un servidor LAMP en una máquina virtual. Deberán configurar Apache para alojar una aplicación web sencilla, gestionar bases de datos en MySQL y crear un script en PHP que interactúe con la base de datos. El entregable será un informe que incluya la documentación del proceso de instalación, configuración y pruebas realizadas.
4. **Criterios cubiertos**: CR1, CR2, CR3, CR5, CR10.
5. **Instrumento de evaluación**: Proyecto.
6. **Duración estimada**: 8 horas.
7. **Recursos necesarios**: Ordenadores con VirtualBox, imágenes de distribución Linux, software Apache, MySQL, PHP, y un editor de código.

### Actividad 2: Monitorización y Optimización de Sistema Operativo
1. **Nombre**: Monitorización y Ajustes del SO
2. **Objetivo**: Comprender cómo monitorizar el rendimiento del sistema operativo y realizar ajustes para optimizar su rendimiento.
3. **Descripción**: En esta actividad, los estudiantes trabajarán en grupos para monitorizar un sistema operativo Linux mediante herramientas como top, htop y vmstat. Deberán identificar procesos que consumen recursos y aplicar configuraciones para mejorar el rendimiento, así como documentar el impacto de dichos ajustes. El entregable será un informe que detalle las acciones realizadas y sus resultados.
4. **Criterios cubiertos**: CR4, CR6, CR7, CR11, CR12.
5. **Instrumento de evaluación**: Práctica.
6. **Duración estimada**: 6 horas.
7. **Recursos necesarios**: Ordenadores con Linux, herramientas de monitorización instaladas, acceso a la red.

### Actividad 3: Gestión de Actualizaciones y Seguridad del Sistema
1. **Nombre**: Actualización y Seguridad del SO
2. **Objetivo**: Aprender a gestionar actualizaciones de software y aplicar medidas de seguridad en un sistema operativo.
3. **Descripción**: En un entorno de laboratorio, los alumnos simularán la administración de un sistema operativo en una empresa. Deberán investigar y aplicar actualizaciones de software, configurar el firewall y establecer políticas de seguridad. Además, realizarán una auditoría de seguridad utilizando herramientas como Lynis. El entregable será un informe que resuma las actualizaciones aplicadas, la configuración de seguridad y los hallazgos de la auditoría.
4. **Criterios cubiertos**: CR8, CR9, CR14, CR15, CR19.
5. **Instrumento de evaluación**: Proyecto.
6. **Duración estimada**: 7 horas.
7. **Recursos necesarios**: Ordenadores con Linux, acceso a internet, herramientas de auditoría de seguridad, y software de gestión de actualizaciones.
