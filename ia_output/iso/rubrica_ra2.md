# Rúbrica — RA2: Gestiona el software de base, aplicando criterios de explotación del sistema.

**Módulo:** Implantación de Sistemas Operativos  
**Ponderación:** 19%

| Criterio | No Alcanzado (0-4) | En Proceso (5-6) | Alcanzado (7-8) | Sobresaliente (9-10) |
|----------|---------------------|-------------------|------------------|-----------------------|
| CR1      | No identifica los componentes de software de base de forma clara. | Reconoce algunos componentes de software de base, pero no todos. | Identifica correctamente todos los componentes de software de base y sus funciones. | Explica detalladamente los componentes de software de base, incluyendo ejemplos de su uso. |
| CR2      | No aplica criterios de explotación del sistema en la gestión del software. | Aplica algunos criterios de explotación, pero con errores. | Aplica criterios de explotación del sistema de manera correcta y coherente. | Aplica criterios de explotación del sistema de forma innovadora y optimizada, mejorando la gestión. |
| CR3      | No realiza la instalación del software de base. | Realiza la instalación con varios errores que afectan su funcionamiento. | Realiza la instalación del software de base de forma correcta y funcional. | Realiza la instalación del software de base de manera excelente, documentando el proceso. |
| CR4      | No gestiona actualizaciones del software de manera adecuada. | Gestiona actualizaciones, pero omite pasos importantes. | Gestiona actualizaciones eficazmente, siguiendo todos los pasos necesarios. | Gestiona y documenta actualizaciones proactivamente, anticipando posibles problemas. |
| CR5      | No configura el software de base según las necesidades del sistema. | Configura parcialmente el software, omitiendo configuraciones clave. | Configura correctamente el software de base ajustándose a las necesidades del sistema. | Configura el software de base de forma avanzada, optimizando el rendimiento del sistema. |
| CR6      | No realiza copias de seguridad del software de base. | Realiza copias de seguridad, pero con errores en el procedimiento. | Realiza copias de seguridad de forma correcta y regular. | Implementa un plan de copias de seguridad completo y documentado, asegurando la integridad de los datos. |
| CR7      | No identifica ni aplica medidas de seguridad en el software. | Reconoce algunas medidas de seguridad, pero no las aplica adecuadamente. | Identifica y aplica medidas de seguridad básicas en el software de base. | Implementa medidas de seguridad avanzadas, evaluando y mejorando la protección del sistema. |
| CR8      | No realiza la gestión de licencias del software de base. | Realiza una gestión de licencias incompleta o incorrecta. | Gestiona adecuadamente las licencias del software de base, cumpliendo con las normativas. | Optimiza la gestión de licencias, proponiendo soluciones para la reducción de costos. |
| CR9      | No realiza la monitorización del rendimiento del software de base. | Realiza monitorización de manera básica, sin análisis significativo. | Realiza una monitorización efectiva del rendimiento, identificando áreas de mejora. | Realiza una monitorización exhaustiva, proponiendo mejoras basadas en análisis profundos. |
| CR10     | No documenta los procedimientos de gestión del software. | Documenta parcialmente, pero falta información clave. | Documenta correctamente los procedimientos de gestión del software de base. | Documenta de manera exhaustiva, proporcionando guías y recomendaciones detalladas. |
| CR11     | No aplica procedimientos de solución de problemas en el software. | Aplica algunos procedimientos, pero con errores que complican la solución. | Aplica procedimientos de solución de problemas de forma efectiva y sistemática. | Desarrolla y aplica soluciones innovadoras a problemas complejos en el software de base. |
| CR12     | No colabora en la gestión del software de forma efectiva. | Colabora, pero de manera limitada y con poca iniciativa. | Colabora activamente en la gestión del software, aportando ideas y soluciones. | Lidera la colaboración en la gestión del software, promoviendo el trabajo en equipo y la comunicación. |
| CR13     | No aplica la normativa vigente sobre gestión del software. | Aplica parcialmente la normativa, con errores de interpretación. | Aplica correctamente la normativa vigente en la gestión del software de base. | Propone mejoras a la normativa y asegura su cumplimiento en todos los aspectos. |
| CR14     | No evalúa la efectividad de las soluciones implementadas. | Realiza evaluaciones superficiales, sin conclusiones claras. | Evalúa efectivamente las soluciones implementadas, proponiendo mejoras. | Realiza evaluaciones exhaustivas y documentadas, implementando cambios estratégicos. |
| CR15     | No realiza la gestión de incidencias relacionadas con el software. | Gestiona incidencias, pero con muchos errores y demoras. | Gestiona incidencias de manera efectiva y en tiempo, manteniendo registros. | Gestiona incidencias proactivamente, implementando mejoras en la gestión del software
