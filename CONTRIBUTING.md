# Contribuir a EvalFP

¡Gracias por tu interés en EvalFP! Este proyecto lo mantiene una profesora de FP y crece con la ayuda de la comunidad docente.

## Formas de contribuir

- **Nuevos módulos de FP**: crea un `*_data.py` en `scripts/modules/` y regístralo en `teacher_config.py` (ver [docs/guia_desarrollo.md](docs/guia_desarrollo.md)).
- **Errores y mejoras**: abre un issue describiendo el problema, con pasos para reproducirlo si aplica.
- **Documentación**: correcciones, ejemplos o traducciones son igual de bienvenidas que el código.
- **Feedback docente**: si usas EvalFP en tu centro, cuéntanos qué falta o qué sobra — el roadmap se prioriza en base a uso real en el aula.

## Flujo de trabajo

1. Haz un fork del repositorio.
2. Crea una rama descriptiva (`feature/modulo-dam`, `fix/calculo-nota-final`).
3. Sigue el estilo de código existente (ver `docs/decisiones_arquitectura.md` para las decisiones de diseño).
4. Abre un pull request explicando el cambio y su motivación.

## Código de conducta

Sé respetuoso/a. Este es un proyecto hecho por y para docentes; el objetivo es ayudarnos mutuamente a reducir carga administrativa.
