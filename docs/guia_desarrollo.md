# Guía de Desarrollo

## Filosofía

Cada Sprint debe dejar el programa completamente funcional.

## Reglas

-   No romper compatibilidad.
-   No usar referencias de celdas fijas.
-   Todo mediante tablas.
-   Todo documentado.
-   Todo probado.

## Flujo de trabajo

Sprint → Implementación → Pruebas → Documentación → Commit → Release
